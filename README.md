# รายวิชา ITC2006 การวิเคราะห์และออกแบบระบบ

## 1. จุดมุ่งหมายรายวิชา
รายวิชานี้มีจุดมุ่งหมายให้นักศึกษามีความรู้และเข้าใจเกี่ยวกับหลักการวิเคราะห์และออกแบบระบบสารสนเทศอย่างเป็นระบบ รวมถึงมีทักษะในการใช้เครื่องมือและเทคนิคต่างๆ เพื่อวิเคราะห์และออกแบบระบบงานธุรกิจ สามารถจัดทำแบบจำลองระบบและออกแบบส่วนประกอบต่างๆ ของระบบสารสนเทศได้ และมีจริยธรรมในการปฏิบัติงานทางด้านเทคโนโลยีสารสนเทศ

## 2. Learning Outcomes
เมื่อสิ้นสุดรายวิชา นักศึกษาจะสามารถ
- อธิบายแนวคิดและวงจรชีวิตของการพัฒนาระบบสารสนเทศและระบบงานธุรกิจได้
- วิเคราะห์ความต้องการและออกแบบระบบสารสนเทศด้วยแนวคิดเชิงวัตถุและ UML
- สร้างแบบจำลองระบบโดยใช้ Use Case, Class Diagram, Activity Diagram และเครื่องมือสนับสนุน
- ประยุกต์ใช้เครื่องมือและวิธีการในการจัดการโครงการพัฒนาระบบสารสนเทศได้
- แสดงออกถึงคุณธรรมและจรรยาบรรณในวิชาชีพด้านเทคโนโลยีสารสนเทศ

## 3. วิธีการจัดการเรียนการสอน
- การบรรยายเนื้อหาทฤษฎีโดยอาจารย์ผู้สอน
- การสาธิตและทำ Workshop เพื่อฝึกปฏิบัติการใช้เครื่องมือวิเคราะห์และออกแบบระบบ
- การอภิปรายกลุ่มและการแก้ไขปัญหากรณีศึกษาจริง
- การประเมินผลผ่านแบบฝึกหัดปฏิบัติการและโครงงาน
- การใช้ซอฟต์แวร์สนับสนุนการออกแบบระบบ (เช่น UML modeling tools)

## 4. หัวข้อที่ครอบคลุมในรายวิชา
- ความรู้เบื้องต้นเกี่ยวกับระบบสารสนเทศและระบบงานธุรกิจ
- การวิเคราะห์ความต้องการระบบ (Requirements Analysis)
- การใช้เครื่องมือ UML ในการออกแบบระบบ (Use Case Diagram, Class Diagram, Activity Diagram)
- การออกแบบฐานข้อมูลและออกแบบส่วนติดต่อผู้ใช้
- วงจรชีวิตและกระบวนการพัฒนาระบบ (SDLC)
- การบริหารจัดการโครงการพัฒนาระบบ
- การทดสอบและการบำรุงรักษาระบบ

## 5. แผนการสอน

| สัปดาห์ | หัวข้อ | เนื้อหา | กิจกรรม |
|---------|---------|----------|----------|
| 1-2 | ความรู้เบื้องต้นเกี่ยวกับระบบสารสนเทศ | ความหมาย ระบบสารสนเทศและองค์ประกอบ | บรรยายและอภิปราย |
| 3-4 | การวิเคราะห์ความต้องการระบบ | วิธีและเทคนิคการเก็บความต้องการ | Workshop, กรณีศึกษา |
| 5-6 | การออกแบบระบบด้วย UML | Use Case Diagram, Class Diagram | สาธิตและปฏิบัติ |
| 7-8 | การออกแบบฐานข้อมูล | ER Diagram, Normalization | Workshop, การบ้าน |
| 9 | สอบกลางภาค | ทบทวนและสอบ | สอบข้อเขียนและปฏิบัติ |
| 10-11 | การออกแบบส่วนติดต่อผู้ใช้ | หลักการออกแบบ UI/UX | สาธิตและอภิปราย |
| 12-13 | วงจรชีวิตและการบริหารโครงการ | SDLC, Project Management | บรรยายและกรณีศึกษา |
| 14-15 | การทดสอบและบำรุงรักษาระบบ | วิธีการทดสอบและเทคนิคบำรุงรักษา | Workshop, โครงงาน |
| 16 | สอบปลายภาค | สอบข้อเขียนและปฏิบัติ | สอบ |

## 6. เกณฑ์การให้คะแนน
- การเข้าชั้นเรียน: 10%
- แบบฝึกหัด / Lab: 20%
- Workshop + งานโครงงาน: 20%
- สอบกลางภาค: 20%
- สอบปลายภาค: 30%
